# Screenshot Courier

定时截取实验电脑屏幕，通过飞书机器人推送到手机，让研究人员不用一直守在仪器旁。

## 下载（无需安装 Python）

在 [Releases](../../releases) 下载 `Screenshot-Courier-v0.1.zip`，解压后双击 `Screenshot Courier.exe` 即可。

## 功能

- **定时截图** — 按设定的间隔自动截取屏幕，JPEG 可调质量
- **飞书推送** — 扫码绑定飞书机器人，截图直接发到飞书消息
- **多人管理** — 支持多个接收人，每人可设独立截图目录
- **邮件兜底** — 飞书失败时自动通过邮件发送
- **本地存档** — 截图按时间命名存本地，支持自动清理
- **阻止息屏** — 运行时阻止电脑息屏/睡眠
- **系统托盘** — 最小化到托盘，后台静默运行

## 使用

### 1. 首次配置

1. 双击运行 `Screenshot Courier.exe`
2. 点击「高级设置 → 接收人」→「新增」→ 填写姓名备注
3. 点击「开始绑定」→用手机飞书扫码，创建飞书智能体应用
4. （可选）填写兜底邮箱：飞书推送失败时改发邮件
5. （可选）填写截图目录：该接收人的截图单独存放

### 2. 日常使用

1. 首页下拉框选择接收人
2. 设置截图间隔（如 15 分钟）
3. 点击「开始监控」
4. 窗口可关闭，程序最小化到右下角托盘继续运行
5. 手机飞书收到截图

### 3. 多人共用

- 每个人首次在该电脑使用时分步扫码一次
- 日常换人实验：首页下拉框切换到另一个人即可
- 管理接收人：设置 → 接收人 → 选择后修改/删除

### 4. 设置说明

| Tab | 内容 |
| --- | --- |
| 接收人 | 管理所有接收人。左侧选择，右侧编辑：名称、飞书重新绑定、兜底邮箱、自定义截图目录、自定义 SMTP |
| 截图 | 全局截图间隔、JPEG 质量、全局截图目录、自动清理策略 |
| 通用 | 开机自启、最小化托盘、阻止息屏、全局 SMTP 发件服务器 |

### 5. 邮件兜底配置

飞书推送失败时自动改发邮件。配置方法：

**全局 SMTP**（所有接收人共用）：设置 → 通用 → 填写发件邮箱/授权码/SMTP 服务器

**个人 SMTP**（该接收人用自己的邮箱发）：设置 → 接收人 → 勾选「使用自定义发件邮箱」→ 填写

常见的 SMTP 服务器：QQ 邮箱 `smtp.qq.com:465`，163 邮箱 `smtp.163.com:465`。

> 授权码不是邮箱密码，需要在邮箱设置中单独生成（QQ 邮箱：设置 → 账户 → POP3/SMTP 服务）。

## 数据存储

- `config.json` — 接收人信息、邮箱、设置等（明文）
- `secrets.dat` — 飞书 app_secret、SMTP 授权码（Windows DPAPI 加密，仅本机当前用户可解密）

文件位于 `%APPDATA%\ScreenshotCourier\`。

## 开发

```bash
pip install -r requirements.txt
python run.py
```

打包：

```bash
pyinstaller "Screenshot Courier.spec"
```

输出在 `dist/Screenshot Courier/`。
